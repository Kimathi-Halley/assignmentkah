content

ANS: a bus is a shared communication pathway that connects
various components of a computer system—such as the CPU, memory,
and input/output (I/O) devices—enabling them to exchange data and control signals.

# 2 Outline similarities and differences between the ISA-bus,
a high-performance bus and the PCI bus.

ANS: Similarities
Both the ISA-bus and PCI-bus serve to connect components by carrying data, addresses, and control signals.
Both Enable system expansion by interfacing various devices.

#3 Describe the benefits of an I/O module and describe the various I/O functions.

Benefits:
Simplified Interface, for a wide variety peripherals
Expandability, I/O is modular and can make it easier to integrate new devices
Buffering, Matcheing the speed of the faster CPU to the slower I/O devices.

I/O functions:
Data Transfer: Moves data between the CPU and peripheral devices.
Address Decoding: Identifies which device should communicate based on address signals.
Signal Conversion: Converts data formats (e.g., digital to analog) as needed.
Control Signal Management: Generates and interprets synchronization and command signals.
Interrupt Handling: Detects and processes asynchronous device signals.
Direct Memory Access (DMA): Facilitates data transfers directly between devices and memory, reducing CPU load.

#4 Distinguish between the three I/O techniques programmes I/O, interrupt-driven I/O and DMA.

Programmed I/O:
- The CPU actively polls the I/O device to check if it’s ready.
- Data transfer is managed directly by the CPU.
- Wastes CPU cycles due to continuous waiting.
- Suitable for low-speed devices with infrequent data transfer.

Interrupt-Driven I/O:
- The device signals the CPU via an interrupt when it is ready.
- The CPU executes an interrupt service routine to handle the I/O.
- Reduces CPU idle time compared to polling.
- More efficient for devices that transfer data at irregular intervals.

Direct Memory Access (DMA):
- A dedicated DMA controller transfers data directly between the I/O device and memory.
- Minimizes CPU involvement, freeing it to perform other tasks.
- Highly efficient for large or high-speed data transfers.
- Requires additional hardware and more complex management.

#5 Describe how the CPU deals with multiple I/O modules if interrupt-driven I/O
is used and how it can determine, which module requested an interrupt.

Multiple I/O modules are connected to a common interrupt controller.
When any module requires service, it sends an interrupt signal to the CPU via this controller.
The interrupt controller either:
• Provides a unique interrupt vector (vectored interrupts), or
• Holds status information that the CPU can poll to determine which device triggered the interrupt.
Using the vector or by reading the controller’s status register, the CPU identifies the source.
It then jumps to the corresponding interrupt service routine to handle the specific I/O module's request.

#6 Recall the characteristics of memory

Memory is characterized by its capacity, speed, volatility, cost, and reliability.
Capacity refers to the total data it can hold.
Speed involves both the access time (latency) and the data transfer rate (bandwidth).
Volatility distinguishes between memory that loses data when power is off lik DRAM and that which retains data (e.g., flash).
Cost per bit and reliability, including error correction features, further define its effectiveness, along with the underlying organization and addressing scheme.

#7 Distinguish between the various access mechanisms to memory, i.e., sequential, direct, random and associative.

Sequential Access:
Data is read in a fixed, predetermined order.
Direct Access:
Each data element is stored at a unique location and can be reached directly by specifying its address.
Random Access:
Any memory location can be accessed independently and with uniform speed.
Associative Access:
data is retrieved based on its content rather than a specific address.

#8 Explain the phrase locality of reference.

Locality of reference refers to the tendency of a program to access a small,
localized set of memory locations repeatedly over a short period.

#9 What is register memory used for?

Register memory is used by the CPU to store operands,
intermediate results, and control data during instruction execution,
providing the fastest possible data access.

#10 What is a cache and where is it located within a computer?

A cache is a small, high-speed memory that temporarily stores frequently accessed data
and instructions to reduce the time required to fetch them from main memory.
It is located on or very close to the CPU, or on the processor chip itself.

#11 Outline how a cache is organised and how it works?

A cache is organized into fixed-size blocks that store copies of data from main memory.
These blocks are arranged in a specific mapping scheme—direct-mapped, set-associative,
or fully associative—which determines how main memory addresses map to cache locations.
HOW IT WORKS: When the CPU requests data, the cache checks for a matching tag to quickly return the data;
if not, the appropriate block is fetched from main memory and stored in the cache.

#12 Distinguish between the cache write policies write through and write back.

Write-Through:
Every write updates both the cache and main memory immediately.
Ensures strong data consistency.
Can incur slower performance due to increased memory traffic.

Write-Back:
Writes update only the cache, marking the block as "dirty."
Main memory is updated only when the dirty block is evicted.
Improves performance by reducing write operations to main memory but requires extra logic for data consistency.

#13 Describe the various mapping functions and replacement algorithms used in a cache.

Mapping Functions: Direct Mapping: Each main memory block maps to one specific cache line. Fully Associative: A memory block can be placed in any cache line. Set-Associative: The cache is divided into sets, and a memory block can be placed in any line within a particular set. Replacement Algorithms: Least Recently Used (LRU): Evicts the block that has not been accessed for the longest period. First-In, First-Out (FIFO): Removes the oldest block in the set based on the order in which blocks were loaded. Least Frequently Used (LFU): Evicts the block that has been accessed the fewest times.

#14 Describe the Program concept.

The program concept in computing refers to a static collection of instructions and data that define a specific computational task.
Written in a programming language, a program is stored in memory and, when executed by the CPU, directs the computer through a series of operations, including control flow and data manipulation.
